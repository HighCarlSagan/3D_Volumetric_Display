⚡ Volumetric POV Display Build Guide

Complete Technical Documentation for Mains-Powered 3D Display

1. How It Works

🧠 The Core Concept

Persistence of Vision (POV): Your brain retains visual information for ~100ms after light stops entering your eye. By spinning an LED matrix at 1200-1800 RPM (20-30 Hz), individual LED flashes merge into a solid 3D image.

How the Display Creates 3D Images:

         LED Matrix (vertical)
              ●●●●●●●●  ← 8 columns
              ●●●●●●●●
              ●●●●●●●●  ← 12 rows
                 ║
                 ║ Spins around central axis
                 ║
         ════════╩════════  Motor shaft
              
As it rotates 360°:
- IR sensor detects reference point once per rotation
- Display shows different "slices" at each angle
- Your eye sees complete 3D cylinder of light

Data Format: Polar coordinates (r, θ, z)
- r = radius (LED position on board)
- θ = angle (changes as board rotates)
- z = height (LED row number)

🔲 LED Matrix

  • 8×12 or 16×12 LEDs
  • Mounted vertically on PCB
  • Forms one radial slice

🔄 Rotation

  • 1200-1800 RPM speed
  • Creates cylindrical volume
  • Motor mounted stationary

📍 Synchronization

  • IR photodiode + reflector
  • Detects once per rotation
  • Triggers image display

⚡ Power

  • Wireless inductive coils
  • 5V @ 300-500mA typical
  • No wires to rotor!

2. Controller Selection: RP2040 vs ESP32

Feature RP2040 (Recommended) ESP32-S3
Simultaneous GPIO ✅ All pins in 1 cycle ❌ Bus contention issues
PIO State Machines ✅ 8 hardware coprocessors ❌ None
Speed 133 MHz (250+ OC) ✅ 240 MHz
WiFi/Bluetooth ❌ None ✅ Built-in
Price ✅ $1-4 $2-3
POV Display Suitability ✅ Excellent ⚠️ Workable but harder

✅ Recommendation: RP2040

Why RP2040 wins: The ability to update all GPIO pins simultaneously is critical for clean POV images. ESP32's bus contention creates timing variations that cause visible ghosting at high speeds.

Best module: Waveshare RP2040-Zero (23×18mm, 29 GPIO, USB-C, ~$4)

💡 What are PIO State Machines?

RP2040's unique feature: 8 programmable I/O coprocessors that can handle precise timing tasks (LED scanning, encoder reading, PWM) independently of the main CPU. Perfect for POV displays where timing is critical.

3. Inductive Power Transfer System

Why Wireless Power?

The rotor spins continuously, so wired connections are impossible. Three options exist:

Method Reliability Pros Cons
Battery High Simple, portable Limited runtime, recharging needed
Slip Rings Medium High current capable Wears out, sparks, noisy
Inductive Coils ⭐ Very High No wear, silent, clean power Limited gap tolerance 
Power System Architecture:

STATIONARY BASE:
AC Mains → 12V PSU → 5V Buck → TX Coil (horizontal)
                                    ║
                            (Magnetic coupling, 2-5mm gap)
                                    ║
ROTATING ASSEMBLY:              RX Coil (horizontal)
                                    ↓
                          Bridge Rectifier (MB6S)
                                    ↓
                           1000µF Smoothing Cap
                                    ↓
                          5V LDO (AMS1117-5.0)
                            ↓              ↓
                      LEDs (5V)    3.3V LDO (AMS1117-3.3)
                                           ↓
                                      RP2040 (3.3V)

Power Budget (8×12 matrix):
- LEDs (scanning): 200-300mA
- RP2040: 50mA
- IR LED: 20mA
- Total: 300-400mA @ 5V ✅ Well within 1A wireless capability

✅ Recommended Setup

Easy option: Adafruit Wireless 5V/1A Charging Coil Set (#1407, ~$10)

  • Built-in regulation on RX side
  • Proven reliability
  • Easy integration
  • Good documentation

⚠️ Alignment is Critical

  • Coils must be concentric (same center)
  • Optimal gap: 3mm (2mm = tight, 5mm = lower efficiency)
  • Even 5mm lateral offset cuts efficiency by 50%
  • Use standoffs or spacers for precise gap

4. Two Design Approaches

Option 1: Matrix Scanning (Recommended)

How Matrix Scanning Works

Instead of 96 pins for 96 LEDs, use 20 pins:

  • 8 column pins (anodes) + 12 row pins (cathodes)
  • Activate one row at a time, set column pattern
  • Scan through all rows at ~10kHz
  • Your eye sees all rows lit due to POV
GPIO Pin Assignment (RP2040):

GPIO 0-7    : LED Columns (8 pins)
GPIO 8-19   : LED Rows (12 pins)
GPIO 20     : IR Position Sensor (input)
GPIO 21     : Motor PWM (output)
GPIO 22     : Status LED (optional)

Total: 21 pins used, 9 still available

LED Wiring:
Each LED anode → Column line
Each LED cathode → Row line
To light LED[Row3, Col5]:
  Set GPIO_ROW3 = LOW
  Set GPIO_COL5 = HIGH
  Current flows: Col5 → LED → Row3 → GND

Pros: Cheapest (~$2 for LEDs), full timing control, fastest updates

Cons: More soldering (96 LEDs), complex PCB routing, careful timing needed

Option 2: Addressable LEDs (APA102)

Smart LEDs with Built-in Drivers

Each APA102 LED contains RGB LED + microcontroller + driver chip

  • Only 2 GPIO pins for unlimited LEDs (Clock + Data)
  • Daisy-chain configuration
  • Built-in current regulation
  • RGB capability for future expansion
Addressable LED Wiring:

RP2040 GPIO 2 (SPI Clock)  ───→ LED[0] → LED[1] → ... → LED[95]
RP2040 GPIO 3 (SPI Data)   ───┘

Each LED:
  VCC → 5V
  GND → GND
  CI ← Clock from previous LED (or RP2040)
  DI ← Data from previous LED (or RP2040)
  CO → Clock to next LED
  DO → Data to next LED

Update Time (96 LEDs at 10MHz SPI): ~314 microseconds
Still fast enough for POV! ✅

Pros: Simple wiring, RGB capable, many GPIO free, built-in current control

Cons: More expensive (~$15 vs $2), higher power, slower protocol

Aspect Matrix Scanning Addressable LEDs
Cost (96 LEDs) ~$2 ~$15
GPIO Pins 20 pins 2 pins
Soldering 96 LEDs + resistors Strip or pre-made
Update Speed Fastest Fast enough
RGB Color No (monochrome only) Yes
Best For Learning, low cost, max control Quick build, future expansion

5. Scaling to 16×12 Configuration

✅ Yes, 16×12 is Achievable!

Doubling to 192 LEDs creates a much more impressive display and is feasible with mains power.

What Changes?

Aspect 8×12 (96 LEDs) 16×12 (192 LEDs)
GPIO Needed 20 pins 28 pins (need custom PCB)
Power (scanning) 250mA avg 400mA avg ✅
Board Width ~40mm ~80mm
Rotational Inertia Baseline 4× higher (need better motor)
Visual Quality Good Excellent!

💡 GPIO Solution for 16×12

Matrix Scanning: 16 columns + 12 rows = 28 GPIO needed

Standard RP2040 modules break out 20-26 GPIO → Need custom PCB with bare RP2040 chip to access all 30 GPIO.

Alternative: Use addressable LEDs (still only 2 GPIO for 192 LEDs!)

⚠️ Mechanical Challenges

  • Balance critical: 80mm wide board needs precise centering
  • More inertia: Takes longer to spin up, need more powerful motor
  • Air resistance: 2× the drag, motor works harder
  • Vibration: Even 1mm offset creates noticeable wobble

Pro tip: Mount heavy components (RP2040, caps) exactly on rotation axis

Recommended Path

  1. Start with 8×12: Prove concept, perfect mechanics, test power system
  2. Design custom PCB: Bare RP2040 chip + all 30 GPIO broken out
  3. Upgrade motor: Brushless recommended for larger rotor
  4. Add balancing: Symmetrical layout + counterweights if needed

💡 Middle Ground: 12×12 Configuration

12 columns + 12 rows = 24 GPIO (fits on RP2040-Zero!)

  • 144 LEDs (50% more than 8×12)
  • Square format (easier to balance)
  • Power: ~300mA (safe)
  • No custom PCB needed

6. Mechanical & PCB Design

Motor Selection

Brushed DC Motor

  • Simple PWM control
  • Very cheap ($2-5)
  • Good for prototyping
  • Electrical noise, wears out

Brushless Motor ⭐

  • Quiet, efficient, long-life
  • Needs ESC controller
  • Best for final build
  • 1806 2300KV ideal size

Target specs: 2000-3000 RPM no-load, 0.5-1.0 N⋅cm torque, <300mA current

PCB Design Strategy

Two-Board Approach (Recommended):

Board 1 - LED Matrix (Vertical):
    ┌─────────────┐
    │  ●●●●●●●●   │ ← 8 or 16 columns
    │  ●●●●●●●●   │
    │  ●●●●●●●●   │ ← 12 rows
    │  (96 LEDs)  │
    │             │
    └─────║║║─────┘
      Edge connector

Board 2 - Controller (Horizontal):
    ┌──────────────────┐
    │  [RP2040] [LDOs] │
    │  [Rectifier]     │
    │                  │
    │  ╔═══════════╗   │ ← RX Coil area
    │  ║   Coil    ║   │   (40-50mm Ø)
    │  ╚═══════════╝   │
    └──────────────────┘
           ║
    Motor shaft passes through

Connected at 90° via headers/edge connector

KiCad Design Tips

  • LED Matrix Board: 2-layer, columns on top, rows on bottom, 1.6mm FR-4
  • Controller Board: 2-layer sufficient for modules, 4-layer for bare RP2040
  • Trace width: 0.5mm for signals, 1-2mm for power
  • Component placement: Heavy parts near rotation axis
  • Mounting holes: For mechanical stability and motor coupling

Balance & Safety

⚠️ Critical Safety Considerations

  • Balance: Test at low RPM first, gradually increase speed
  • Enclosure: Clear acrylic case to contain debris if something breaks
  • Emergency stop: Accessible kill switch
  • Secure mounting: Motor firmly attached, shaft properly coupled
  • Eye protection: When testing at full speed

7. Recommended Build Phases

Phase 1: Static Testing

  • Build LED matrix on breadboard
  • Test RP2040 driving LEDs (no rotation)
  • Verify matrix scanning or addressable LED control
  • Display simple test patterns

Phase 2: Power System

  • Set up inductive power transfer
  • Mount TX coil on base, RX on test platform
  • Verify stable 5V/3.3V while rotating by hand
  • Test with LED matrix powered

Phase 3: Mechanical Assembly

  • Design & fabricate PCBs in KiCad
  • Assemble LED matrix + controller boards
  • Mount on motor shaft with proper coupling
  • Balance rotor assembly carefully

Phase 4: Position Sensing & Speed Control

  • Install IR LED and photodiode
  • Add reflective target on base
  • Implement motor speed control (PWM)
  • Test synchronization at low speed

Phase 5: Software & Content

  • Write display firmware (matrix scanning or SPI)
  • Implement position-triggered display updates
  • Create volumetric data pipeline (Blender → binary)
  • Test with simple 3D shapes, then complex animations

9. Quick Decision Guide

✅ Recommended Starting Configuration

Controller: RP2040 (Waveshare RP2040-Zero)

LEDs: 8×12 matrix scanning (96 discrete 0805 LEDs)

Power: Adafruit wireless charging coils

Motor: Small brushed DC for prototype, upgrade to brushless later

Why: Proven design, lowest cost, maximum learning, good performance

If You Want... Choose This
Lowest cost, maximum control 8×12 matrix scanning with RP2040
Quick build, simple wiring 8×12 addressable LEDs (APA102)
Best visual quality 16×12 matrix scanning (custom PCB)
RGB color capability Addressable APA102 LEDs
Easiest first project 8×12 matrix, pre-made modules, breadboard prototype first

Key Success Factors

  1. Balance is critical: Spend time getting rotor perfectly centered
  2. Start slow: Test at 300 RPM before going to 1800 RPM
  3. Power first: Prove wireless power works before adding complexity
  4. Incremental testing: Each phase should work before moving on
  5. Safety always: Enclosure, emergency stop, eye protection

Good luck with your volumetric POV display build! Start with the basics, test thoroughly, and scale up gradually.
